# 内置菜谱与 Kotlin 编译配置设计

## 目标
1. 让 App 在首次运行时自动填充 7 类（Breakfast/Lunch/Dinner/Dessert/Snack/Soup/Salad）各 20 条示例菜谱数据（共 140 条），数据包含现有字段加 `tags`、`nutritionalNotes`、`preparationTips`。
2. 持久化这些字段，方便 UI（如详情页）展示更多上下文信息。
3. 避免 Room 表结构冲突，同时升级 Kotlin 编译配置到 `compilerOptions` DSL，消除 AGP 8.7/Gradle 9 的弃用警告。

## 数据资产与结构
- 在 `res/raw/good_recipe_seed.json` 里放一份 JSON 数组，140 条对象每条含 `title/description/ingredients/steps/category/cookTimeMinutes/servings/imageUrl/isFavorite/tags/nutritionalNotes/preparationTips`。
- 以 `category` 字符串记录 `RecipeCategory` 的 `displayName`（中文名），`tags` 为字符串数组，如 `["家常", "低脂"]`。
- 借助脚本从几个模板快速生成所有条目，确保内容可扩展。

## 数据载入与注入
- 新建 `SeedDataProvider`（在 `data/seed` 包）负责读取 `good_recipe_seed.json`、转换成域模型里的 `Recipe` 对象，维护字符串<->列表的序列化逻辑。
- 添加 `SeedDataInitializer`（`@Singleton`，`@Inject` `RecipeRepository` + `Context`）包含 `suspend fun seedIfNeeded()`：
  1. 通过 `repository.getAllRecipes().firstOrNull()` 判断是否已有数据。
  2. 若为空，则调用 `SeedDataProvider.loadRecipes(context)` 并按批次 `insertRecipe`。可以用 `withContext(Dispatchers.IO)` 或 `repository` 内协程，但要避免重复插入。
- 在 `GoodRecipeApp.onCreate()`（`@HiltAndroidApp`）用 `CoroutineScope(Dispatchers.IO).launch { seedDataInitializer.seedIfNeeded() }` 或 `lifecycleScope` 触发初始化。

## Schema & Mapping
- `RecipeEntity` 增加三列：`tags`（text, 用 `||` 连接）、`nutritionalNotes`、`preparationTips`（均 `TEXT NOT NULL DEFAULT ''`），并把 `@Entity` 表版本从 1 提升到 2。
- 在 `AppModule` 的 `Room.databaseBuilder()` 添加 `addMigrations(MIGRATION_1_2)`，用于新增列。
- 转换函数 `RecipeEntity.toDomain()` 与 `Recipe.toEntity()` 更新，确保 `tags` 通过 `.split("||")`/`.joinToString("||")` 保留顺序，新字段一并映射。
- `Recipe` 数据类增加 `tags: List<String> = emptyList(), nutritionalNotes: String = "", preparationTips: String = ""`，并传播至所有引用（ViewModel/DAO 未需变更）。

## UI/展示建议（可选）
- `RecipeDetailScreen` 里加入如下分区：
  - `ChipGroup` 展示 `tags`，`nutritionalNotes` 与 `preparationTips` 作为轻量文本，只有非空时才渲染。
- 其余 UI 继续使用已有 `Recipe` 结构，不受影响。

## Kotlin 编译配置
- 将 `app/build.gradle.kts` 的 `kotlinOptions { jvmTarget = "17" }` 换成 `compilerOptions { kotlinOptions.jvmTarget.set("17") }`，消除 Gradle 9 的警告。未来需要使用 `compilerOptions.freeCompilerArgs`（如有）也放在新 DSL 里。

## 验证办法
1. 运行 `./gradlew :app:assembleDebug`，确保编译（含 seed 相关代码）通过。
2. 运行 App，在数据库为空时检查 `recipes` 表是否已插入 140 条记录（通过日志 `seedDataInitializer` 打点等）。
3. 查看 `RecipeDetailScreen`（如更新）确保 `tags`/`notes` 正确渲染。

## 风险与对策
- Room Schema 升级需 Migration，漏记会导致运行时崩溃：通过 `Migration(1, 2)` 添加列并设置默认值。
- JSON 解析异常：`SeedDataProvider` 应捕获 `JSONException`，日志提示，并在无法插入时避免阻塞 App 启动。
- 文件太大：把 JSON 放在 `raw/`，没有压缩器问题，构建体积增加可接受。
