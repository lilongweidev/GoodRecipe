# GoodRecipe 内置菜谱扩充与编译配置设计

## 目的
1. 保证 App 启动时内置的 7 类（早餐/午餐/晚餐/甜点/小吃/汤品/沙拉）各有 20 道真实感菜谱，140 条数据里包含 `title`、`description`、`ingredients`、`steps`、`category`、`cookTimeMinutes`、`servings`、`imageUrl`、`isFavorite`、`tags`、`nutritionalNotes`、`preparationTips`，方便 UI 展示丰富信息。
2. 把这些数据组织成可维护的生成器：早餐保持现有手工定义，其余 6 类通过可配置的前缀+核心组合自动派生，避免手动敲 120 条重复结构。
3. 将 Kotlin 编译块从旧 DSL (`kotlinOptions`) 迁移到 Gradle 9 推荐的 `compilerOptions`，消除 AGP 8.7.3/Gradle 9 的 warn，保持 future-proof。

## 数据生成策略
- **静态部分**：保留现有脚本里那 20 条早餐样本，继续作为 `category_recipes["早餐"]`。
- **动态部分**：定义一个 `Blueprint` 数据结构，包含
  1. `prefixes`（如 `蜜香`、`烟熏` 等）附带 `slug` 片段、额外调味料、味型提示；
  2. `cores`（如「奶油野菌意面」「香料番茄炖猪肉」）记录主要食材、佐料、标签、预设 `cookTime`/`servings`、`prepActions` 模板、`nutritionalNotesTemplate` 与 `preparationTipsTemplate`。
- 通过遍历 `prefix × core` 组合（每类 5 个 core × 4 个 prefix），生成 20 条菜谱：
  - `title` = `{prefix.text}{core.name}`；
  - `slug` = `{category_slug}-{prefix.slug}-{core.slug}`，供 `imageUrl` 和调试使用；
  - `description`/`steps` 由模板函数格式化，确保每条文字都提及 prefix 与核心亮点。
  - `ingredients` 由 `core.ingredients` 加上 prefix 指定的 `extraIngredient`，去重；
  - `tags`、`nutritionalNotes`、`preparationTips` 也由模板拼出，保证 `tags` 包含>1 项，`nutritionalNotes` 提及主食材。
- 每类 `Blueprint` 内容如下（示例）：
  - 午餐：柠香蒜烤鸡、奶油野菌意面、墨西哥牛肉碗、地中海烤菜藜麦、香料番茄炖猪肉。
  - 晚餐：红酒炖牛腩、椰香咖喱虾、铁板黑椒羊排、韩式辣酱五花、香柠烤鳕鱼（各配 prepActions 等）。
  - 甜点/小吃/汤品/沙拉：同样定义 5 个核心，每个搭配 4 个风味前缀，例如甜点前缀含“焦糖”“轻盈”“绵密”“荔枝”，核心涵盖布丁、挞、蛋糕。
- `generate_dynamic_entries(category, blueprint)` 负责生产符合现有脚本格式的字典（可含 `actions` 或 `steps`），最后 `category_recipes = {"早餐": breakfast_entries, **dynamic_entries}`。
- 依然使用 `build_recipe` 处理通用字段，统一 `imageUrl` 模板为 `https://example.com/images/{slug}.jpg`。

## 数据注入 & seed
- 新增 `data/seed/SeedDataProvider.kt`（读取 `res/raw/good_recipe_seed.json`，返回 `List<Recipe>` 表示），使用 `InputStream` + `Json` 解析。
- 新增 `SeedDataInitializer`（`@Singleton`）：在 `seedIfNeeded()` 里先调用 `repository.getAllRecipes().firstOrNull()`，若为空，逐条 `insertRecipe`，必要时包裹 `withContext(Dispatchers.IO)`；`isFavorite` 默认为 `false`。
- 在 `GoodRecipeApp.onCreate()` 通过 `CoroutineScope(Dispatchers.IO).launch { seedDataInitializer.seedIfNeeded() }` 启动，避免主线程阻塞。

## Kotlin 编译配置
- 在 `app/build.gradle.kts` 的 `android` 模块里，把原先的 `kotlinOptions { jvmTarget = "17" }` 替换为：
  ```kotlin
  compilerOptions {
      kotlinOptions.jvmTarget.set("17")
  }
  ```
- 若未来需要 `freeCompilerArgs`（目前无），也统一放在 `compilerOptions` 内。

## 验证
1. `./gradlew :app:assembleDebug`（检查脚本生成的 JSON + 新生 seed 逻辑是否编译通过）。
2. 运行 App（或 `adb shell` 查看 Room 表）验证第一次启动后 `recipes` 表有 140 条记录。
3. 打开任意详情页，确认 `tags`/`nutritionalNotes`/`preparationTips` 没有空值。

## 备注
- 由于 sandbox 无法直接从 TheMealDB API 抓取数据（`curl`/`web.run` 均未返回 JSON），脚本改为以可配置前缀+核心组合生成内容，但仍借助你提供的早餐 json 做为样板，确保所有字段格式一致。
- 跨类别组合减少重复工作，也方便将来补充更多 prefix/core。
